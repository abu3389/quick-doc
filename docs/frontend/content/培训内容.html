  • 如果 npm install指令安装失败，可以尝试使用cnpm install 或者 yarn install 进行安装，前提是cnpm或者yarn指令已被安装

npm install cnpm -g

或者

npm install yarn -g
  • 如果这样安装还是提示cnpm 或 yarn 不是内部命令，需要配置环境变量：

请根据全局安装淘宝镜像后给出的路径提示

# 项目目录详解

解读目的：

解读项目目录是为了对项目框架内所建设的已有资源和工具加以合理利用提高编码效率，建议先了解目录作用，再通过实践运用。

下一章节将会重点介绍一些针对医保业务使用的工具类用法。

需要注意部分项目目录因开发人员不同结构有会有所差异，但大同小异。

这里以网厅项目为例介绍目录用途：

# 根目录讲解

  • .scannerwork 存放sonar插件扫描后的结果文件

  • .vscode 编辑器格式化配置文件（仅在vscode有效，建议使用vscode编辑，避免编辑器格式化配置不一样导致代码冲突）

  • build 自动化打包部署脚本文件夹，运行文件夹里的publishCompress.sh脚本后一键生成腾讯部署压缩包

  • dist 存放普通打包方式打包生成出来的前端文件

  • node_modules 项目依赖包（即npm install 指令下载的文件）

  • public 存放公共静态文件（该文件夹里的文件打包时不会被编译，原样输出，可以保证项目的全局配置文件如：project.config.js在项目打包后还可以进行配置，方便打包下发地方可以应对不同环境需求的配置）

  • src 编写代码的主要存放文件夹（下面会单独拿出来介绍）

  • .env.prod 生产环境可以拿到的一些变量，仅在生产环境才会读取，即npm run serve，可通过package.json配置脚本使用的环境（主要用于区分不同运行环境做不同的事情）

  • .env.proxy 测试环境可以拿到的一些变量，仅在测试环境才会读取，即npm run build:prod，可通过package.json配置脚本使用的环境（主要用于区分不同运行环境做不同的事情）

  • .eslintignore 如果开启Eslint严格模式语法校验（默认未开启），Eslint要忽略校验的文件夹

  • .eslintrc.js 如果开启Eslint严格模式语法校验（默认未开启），Eslint的校验自定义规则配置文件

  • .gitignore git上传要忽略的文件夹或文件

  • babel.config.js babel语法转换配置（主要用来做浏览器对ES6/ES7的语法兼容），可以参考配置：配置 Babel · Babel 中文文档 (opens new window)

  • jsconfig.json 如果一个项目没有显式的使用 require 或者 import 引用库，也没有在package.json里声明这些库的js，那么 VSCode 并不会在当前项目中对这个库进行智能提示。 可以参考配置：Visual Studio Code 下的 jsconfig.json (opens new window)

  • package.json 依赖包模块的描述文件（用于配置运行脚本指令和依赖包安装项）

  • README-zh .md 项目中文文档介绍（拉取项目下来一般可以看下这个介绍，有助于更快了解项目和注意事项）

  • sonar-project.properties sonar 扫描插件配置文件

  • package.json 依赖包模块的描述文件（用于配置运行脚本指令和依赖包安装项）

  • vue.config.js 项目脚手架的配置文件（可用于配置跨域，配置webpack分块打包，输出目录，编译规则等等）

# src目录详解：

  • api 存放后端接口的文件夹

  • assets 存放静态文件的目录（一般是图片等等）

  • components 存放自定义组件库（需要被多个页面多次用到的组件可以放在这里）

  • data 存放前端本地静态JSON数据（如城市区划json数据等等）

  • env 存放静态全局变量的配置文件

  • icons 存放字体图标的文件夹，可通过iconfont (opens new window)获取svg图标

  • layout 存放支撑整个项目的骨架文件（包括左侧菜单，顶部导航栏，面包屑，显示的页面的容器和底部等等）

  • router 存放页面路由及路由配置文件

  • store 全局状态管理，存放可动态联动改变的全局变量、操作动态全局变量的方法等等

  • styles 存放全局样式文件

  • theme 存放UI组件的主题定制样式文件

  • utils 一些封装好的工具函数，引入即可使用

  • views 路由对应的展示页面，一般配合自定义组件使用

  • App.vue 整个项目的展示容器页，通过里面的router-view配合路由访问前端页面

  • main.js 项目的主入口文件，所有需要的依赖包，工具等等一切支撑和辅助脚手架项目运行全局初始化的准备工作都在这里生成和挂载

  • permission.js 权限拦截器，用于页面鉴权，判断登录状态等等

  • polyfill.js 浏览器es6/es7语法兼容补丁

  • settings.js 骨架的一些配置项（项目标题、菜单是否伸缩，是否展示logo等等）

  • use.js 用于自定义组件的全局注册，每次使用自定义组件无需再引入（适用于使用频次较多的自定义组件：如头部、尾部、标题组件等等）

# filters 全局过滤函数

# 全局注入：
src/main.js

//引入全局过滤函数（字段过滤转换）
import * as filters from '@/utils/filters' 

// 全局过滤器注册
Object.keys(filters).forEach(key => {
  Vue.filter(key, filters[key])
})

src/utils/filters.js

//手机号脱敏处理
export const mobile = (mobile) => {
  if (!mobile) {
    return '-'
  }
  return String(mobile).substr(0, 3) + '****' + String(mobile).substr(7, 4)
}

/**
 * 时间戳日期过滤
 * @param {*} time 时间戳
 * @param {*} cFormat 自定义时间格式 参数: y年 m月 d日 h时 i分 s秒 a星期 格式如：'{y}-{m}-{d} {h}:{i}:{s}'
 */
export const dataFormat = (time, cFormat) => {
  //判断是否传空
  if (typeof time === "object") {
    return "-"
  }
  //判断是否已转换过了
  if (typeof time === "string" && time.length > 0) {
    return time
  }
  //没有再去转换
  const res = parseTime(time, cFormat)
  const data = res.indexOf("NaN") > -1 ? "-" : res
  return data
}

// 表格空值格式化
export const empty = (data) => {
  return data || "-"
}

// 金额过滤（精确到小数点后三位）
export const money = (data) => {
  if (!data) {
    return '-'
  }
  return Number(data).toFixed(3).replace(/\d$/, '')
}

... 更多方法省略，可关注每个项目文件中的具体方法。

# mixins 全局混入函数

# 全局注入：
src/main.js

//公共函数

// 公共函数注册
Vue.mixin(comMixins)

src/utils/comMixins.js

export default {
  data() {
    return {
       //此data内的变量会与每个.vue文件中的data变量进行合并。
    }
  },
  created() {
    //此此周期函数钩子会与每个.vue文件的created生命周期合并执行。
  },
  mounted() {
    //此周期函数钩子会与每个.vue文件的mounted生命周期合并执行。
  },
  methods: {
    /**
     * 表格空值格式化
     */
    CellEmpty(row, column, cellValue, index) {
      return this.$options.filters.empty(cellValue) //empty方法查看上面的filters.js
    },
    /**
     * 表格金额格式化
     */
    CellMoney(row, column, cellValue, index) {
      return this.$options.filters.money(cellValue) //money方法查看上面的filters.js
    },
    /**
     * 表单重置公共方法
     * @formName 表单的ref名称 可以为数组 或者字符串
     * 如："editForm"、["editForm1","editForm2"]
     */
    $resetFormFields(formName) {
      if (isEmpty(formName)) return
      if (formName.constructor == String) {
        if (!this.$refs[formName]) return
        this.$refs[formName].resetFields()
      } else if (formName.constructor == Array) {
        for (let i = 0, l = formName.length; i < l; i++) {
          if (!this.$refs[formName[i]]) continue
          this.$refs[formName[i]].resetFields()
        }
      }
    },
    ... 更多方法省略，可关注每个项目文件中的具体方法。
  }
}

// 校验身份证
export const idcard = checkWrap(value => checkIDCardNumber(value), '请输入正确的身份证号码')

// 校验文本长度
export const textLength = (length = 0) => {
  return checkWrap(value => value.length <= length, `字段长度不可大于${length}`)
}

// 生成基于element表单校验规则中 validator 的函数格式
// 详见：https://element.eleme.cn/#/zh-CN/component/form#zi-ding-yi-xiao-yan-gui-ze
function checkWrap(checkFunction, error) {
  return (rule, value, callback) => {
    if (!value) {
      return callback()
    }
    if (!checkFunction(value)) {
      return callback(new Error(error))
    }
    callback()
  }
}
