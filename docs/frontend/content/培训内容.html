# 项目目录详解

解读目的：

解读项目目录是为了对项目框架内所建设的已有资源和工具加以合理利用提高编码效率，建议先了解目录作用，再通过实践运用。

下一章节将会重点介绍一些针对医保业务使用的工具类用法。

需要注意部分项目目录因开发人员不同结构有会有所差异，但大同小异。

这里以网厅项目为例介绍目录用途：

# 根目录讲解

  • .scannerwork 存放sonar插件扫描后的结果文件

  • .vscode 编辑器格式化配置文件（仅在vscode有效，建议使用vscode编辑，避免编辑器格式化配置不一样导致代码冲突）

  • build 自动化打包部署脚本文件夹，运行文件夹里的publishCompress.sh脚本后一键生成腾讯部署压缩包

  • dist 存放普通打包方式打包生成出来的前端文件

  • node_modules 项目依赖包（即npm install 指令下载的文件）

  • public 存放公共静态文件（该文件夹里的文件打包时不会被编译，原样输出，可以保证项目的全局配置文件如：project.config.js在项目打包后还可以进行配置，方便打包下发地方可以应对不同环境需求的配置）

  • src 编写代码的主要存放文件夹（下面会单独拿出来介绍）

  • .env.prod 生产环境可以拿到的一些环境变量，仅在生产环境才会读取，即npm run serve，可通过package.json配置脚本使用的环境（主要用于区分不同运行环境做不同的事情）

  • .env.proxy 测试环境可以拿到的一些环境变量，仅在测试环境才会读取，即npm run build:prod，可通过package.json配置脚本使用的环境（主要用于区分不同运行环境做不同的事情）

注意：

vue脚手架项目自带环境变量 NODE_ENV 和 BASE_URL 等。

全局变量可以用 process.env.变量名称 获取，例如：process.env.BASE_URL。

若想在.env文件添加自定义全局变量，请使用 VUE_APP_变量名称 的命名，例如：VUE_APP_UNIT，否则获取不到全局变量。

  • .eslintignore 如果开启Eslint严格模式语法校验（默认未开启），Eslint要忽略校验的文件夹

  • .eslintrc.js 如果开启Eslint严格模式语法校验（默认未开启），Eslint的校验自定义规则配置文件

  • .gitignore git上传要忽略的文件夹或文件

  • babel.config.js babel语法转换配置（主要用来做浏览器对ES6/ES7的语法兼容），可以参考配置：配置 Babel · Babel 中文文档 (opens new window)

  • jsconfig.json 如果一个项目没有显式的使用 require 或者 import 引用库，也没有在package.json里声明这些库的js，那么 VSCode 并不会在当前项目中对这个库进行智能提示。 可以参考配置：Visual Studio Code 下的 jsconfig.json (opens new window)

  • package.json 依赖包模块的描述文件（用于配置运行脚本指令和依赖包安装项）

  • README-zh .md 项目中文文档介绍（拉取项目下来一般可以看下这个介绍，有助于更快了解项目和注意事项）

  • sonar-project.properties sonar 扫描插件配置文件

  • package.json 依赖包模块的描述文件（用于配置运行脚本指令和依赖包安装项）

  • vue.config.js 项目脚手架的配置文件（可用于配置跨域，配置webpack分块打包，输出目录，编译规则等等）

# src目录详解

  • api 用于存放后端接口配置

  • assets 存放静态文件的目录（一般是图片等等）

  • components 存放自定义组件库（需要被多个页面多次用到的组件可以放在这里）

  • data 存放前端本地静态JSON数据（如城市区划json数据等等）

  • env 存放静态全局变量的配置文件

  • icons 存放字体图标的文件夹，可通过iconfont (opens new window)获取svg图标

  • layout 存放支撑整个项目的骨架文件（包括左侧菜单，顶部导航栏，面包屑，显示的页面的容器和底部等等）

  • router 存放页面路由表及路由配置

  • store 全局状态管理，存放可动态联动改变的全局变量、操作动态全局变量的方法等等

  • styles 存放全局样式文件

  • theme 存放UI组件的主题定制样式文件

  • utils 存放一些封装好的工具函数，部分已全局引入，少部分手动引入即可使用。

  • views 路由对应的展示页面，一般配合自定义组件使用

  • App.vue 整个项目的展示容器页，通过里面的router-view配合路由访问前端页面

  • main.js 项目的初始化主入口文件，所有需要的依赖包，工具等等一切支撑和辅助脚手架项目运行全局初始化的准备工作都在这里生成和挂载

  • permission.js 文件用于页面鉴权，判断登录状态等

  • polyfill.js 浏览器es6/es7语法兼容补丁

  • settings.js 骨架的一些配置项（项目标题、菜单是否伸缩，是否展示logo等等）

  • use.js 用于自定义组件的全局注册，每次使用自定义组件无需再引入（适用于使用频次较多的自定义组件：如头部、尾部、标题组件等等）

src/utils/filters.js

//手机号脱敏处理
export const mobile = (mobile) => {
  if (!mobile) {
    return '-'
  }
  return String(mobile).substr(0, 3) + '****' + String(mobile).substr(7, 4)
}

/**
 * 时间戳日期过滤
 * @param {*} time 时间戳
 * @param {*} cFormat 自定义时间格式 参数: y年 m月 d日 h时 i分 s秒 a星期 格式如：'{y}-{m}-{d} {h}:{i}:{s}'
 */
export const dataFormat = (time, cFormat) => {
  //判断是否传空
  if (typeof time === "object") {
    return "-"
  }
  //判断是否已转换过了
  if (typeof time === "string" && time.length > 0) {
    return time
  }
  //没有再去转换
  const res = parseTime(time, cFormat)
  const data = res.indexOf("NaN") > -1 ? "-" : res
  return data
}

// 表格空值格式化
export const empty = (data) => {
  return data || "-"
}

// 金额过滤（精确到小数点后三位）
export const money = (data) => {
  if (!data) {
    return '-'
  }
  return Number(data).toFixed(3).replace(/\d$/, '')
}
